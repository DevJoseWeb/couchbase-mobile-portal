This lesson teaches you to

  1. ターミナルを開く
  2. プロジェクトの作成
  3. サンプルコードのソースコードをコピーしましょう。
  4. プロジェクトの中にCouchbase Liteのプラグインを追加しましょう。
  5. プロジェクトの中に他のプラグインも追加しましょう。
  6. iOSサンプルアプリを起動
  7. Androidサンプルアプリを起動

ビルドと実行

これでTodo Liteアプリを作るための全てのツールをインストールしました。もし問題があれば、トラブルシューティングを確認してください。

さて、タイマーを出して実際に５分間でビルドできるかやってみましょう！

ターミナルを開く

ターミナルを開いてTodo Liteアプリを保存したいディレクトリに変更しましょう。

プロジェクトの作成

新規にPhoneGapのプロジェクトを作成し、そこにディレクトリを作りましょう：

$ phonegap create todo-lite com.couchbase.TodoLite TodoLite
$ cd todo-lite

PhoneGapでプロジェクトを作成した後、ディレクトリ名にwwwが付いてきます。このwwwは、ディレクトリがデフォルトのアプリが含まれていることを指します。

サンプルコードのソースコードをコピーしましょう。

デフォルトのアプリをTodo Liteのソースコードと一緒にwww の中に入れましょう：

$ rm -rf www
$ git clone https://github.com/couchbaselabs/TodoLite-PhoneGap.git www

ほとんどのアプリケーションコードはindex.jsファイルから確認できます。

独自でアプリケーションを作りたい場合、デフォルトのアプリを自分のソースコードと一緒にwwwのディレクトリに保存しましょう。

プロジェクトの中にCouchbase Liteのプラグインを追加しましょう。

$ phonegap plugin add https://github.com/couchbaselabs/Couchbase-Lite-PhoneGap-Plugin.git

プロジェクトの中に他のプラグインも追加しましょう。

下記のプラグインはいくつかの機能についてサンプルアプリで使われています。アプリは下記のを使用しなくても十分機能しますので、必要ない場合は追加しなくても問題ありません。

$ phonegap plugin add org.apache.cordova.camera
$ phonegap plugin add org.apache.cordova.inappbrowser
$ phonegap plugin add org.apache.cordova.network-information

これらのプラグインはInAppBrowser(カメラ)とネットワーク情報を起動します。

iOSサンプルアプリを起動

Todo Lite iOSアプリをビルドして起動します：

$ phonegap run ios

Todo LiteアプリはiOSのシミュレーターで開きます。iOSのシミュレーターが自動的に開かない場合、ios-simがインストールされているか確認しましょう。

Androidサンプルアプリを起動

Todo Lite Androidアプリをビルドして起動します：

$ phonegap run android

Todo LiteアプリはAndroidのシミュレーターで開きます。Androidのシミュレーターが開かず、Adobe PhoneGapのビルドサービスにログインされるようメッセーが表示された場合、エミュレーターの設定に問題がある可能性があります。このサンプルアプリはAdobe PhoneGapのビルドサービスにログインせずに開けますので、改めて設定の確認をしてみましょう。
