POST /{db}/_purge

In this document

  • リクエスト
  • レスポンス
  • 例

このリクエストは永久にデータベースから外します。

ドキュメントを削除する時にはデータベースから消えません。他のデータベースに同期できるために削除されたというフラッグを入れ、 新規リビジョンを作成するだけです。さらに、このやり方によってドキュメントが削除されたいるかどうかチェックすることができます。

パージによってデイスクスペースが戻って来るわけではありません。デイスクスペースを取り戻すためにデータベースの圧縮（/db/_compact)、 もしくはビューの圧縮（/db/_compact/design-doc)を行わなければなりません。

Caution:

POST /db/_purgeは予想外の副作用を持つ場合もありますので、あまり使わない方がよいでしょう。

リクエスト

リクエストのヘッダー

このリクエストに必要なヘッダーがありません。

クエリパラメーター

このリクエストは以下のパラメーターをしようします：

名前種類説明デフォルト値
n/aオブジェクトパージされるドキュメントを表せるオブジェクト。キーはドキュメントIDでバリューはパージされるリビジョンIDの配列になる。

レスポンス

ステータスコード

  • 200 OK – リクエストが正常に終わった
  • 400 Bad Request – 無効なデータベース名か無効なJSON
  • 415 Unsupported Media Type – 無効なContent-Typeヘッダー

レスポンスヘッダー

このレスポンスは普段のHTTPヘッダーしか使いません。

メッセージボディー

レスポンスのメッセージボディーは以下のオブジェクトが入っているJSONドキュメントを入れます：

名前種類説明
purge_seq　整数パージの連番号
purgedオブジェクトパージされたドキュメントを表せるオブジェクト。キーはドキュメントIDでバリューはパージされるリビジョンIDの配列になる。

例

リクエスト

POST /cookbook/_purge HTTP/1.1
