DELETE /{db}/_user/{name}/ _session/{session-id}

In this document

  • Request
  • Response
  • Example

This request deletes a single session for a specified user. If the session is not assigned to the named user, the session is not deleted.

Request

Request headers

This request does not have any required headers.

Query parameters

This request does not use query parameters.

Message body

This request does not use a message body.

Response

Status codes

  • 200 OK – Request completed successfully
  • 404 Not Found – Specified session not found for user

Response headers

This response uses only standard HTTP headers.

Message body

If the session is successfully deleted, the response has an empty message body. If the session is not deleted, the message body contains error information.

Example

The following example deletes a session with session ID 6f03152a5b1cac1e3cf2f15f4f68e8f5202cb876, belonging to user chef123 in database named cookbook.

Request

DELETE /cookbook/_user/chef123/_session/6f03152a5b1cac1e3cf2f15f4f68e8f5202cb876 HTTP/1.1
Host: localhost:59840

Response

HTTP/1.1 200 OK
Content-Length: 98
Content-Type: application/json
Date: Thu, 06 Nov 2014 00:14:25 GMT
Server: Couchbase Sync Gateway/unofficial

The following example shows the response returned when you attempt to delete a session that does not exist on the server.

Request

DELETE /cookbook/_user/chef123/_session/1234 HTTP/1.1
Host: localhost:59840

Response

HTTP/1.1 404 Not Found
Content-Length: 40
Content-Type: application/json
Date: Thu, 06 Nov 2014 00:30:09 GMT
Server: Couchbase Sync Gateway/unofficial
{
    "error": "not_found", 
    "reason": "missing"
}
