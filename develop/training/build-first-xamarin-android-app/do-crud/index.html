Performing CRUD operations

CRUD operations represent the basic interactions with a Couchbase Lite database. This section demonstrates how to create, retrieve, update, and delete documents. The tutorial code creates just one document and then in turn retrieves it, updates it, and finally deletes it.

In this section, you add code to the MainActivity class.

Create a document

This section of the tutorial code generates some data for a new document, and then creates the new document and writes it to the database. It also outputs some messages to the console.

  1. Add the following method to to create a document, and call it from the HelloCBL method: 

    string CreateDocument ()
{
    var doc = db.CreateDocument ();
    string docId = doc.Id;
    var props = new Dictionary<string, object> {
        { "name", "Big Party" },
        { "location", "MyHouse" },
        { "date", DateTime.Now }
    };
    try {
        doc.PutProperties (props);
        Log.Debug (TAG, String.Format("doc written to database with ID = {0}", doc.Id));
    } catch (Exception e) {
        Log.Error (TAG, "Error putting properties to Couchbase Lite database", e);
    }
    return docId;
}

    void HelloCBL ()
{
    try {
        db = Manager.SharedInstance.GetDatabase (DB_NAME);
    } catch (Exception e) {
        Log.Error (TAG, "Error getting database", e);
        return;
    }
    // Create the document
    CreateDocument ();
}

    Dictionary objects provide JSON-compatible representations of data that are suitable for creating documents that can be stored in the database. The content for the document created by this snippet is created in a Dictionary object named props that contains these keys: name and location. Each key has a value associated with it. The value for the name key contains the string “Big Party”, and the value for the location key contains the location the event is held at.

    The code snippet creates an empty Document object named doc that is saved to the database by using the Document class PutProperties() method. The PutProperties method can throw an exception, so the call is placed in a try block. The PutProperties method writes the contents of the dictionary into the Document, which actually saves the passed in Dictionary to the Couchbase Lite database on the local device. We return the documentID so that the caller has the document ID for their records.

    When a document is saved to the database, Couchbase Lite generates a document identifier property named _id and a revision identifier property named _rev, which are added to the stored document.

  2. Build and run the app.

  3. View the Application Output.

    The Application Output displays a message that logs the initial document content and the identifier that was assigned to it upon creation.

    [CouchbaseEvents] doc written to database with ID = b7c49d1e-4b24-491e-b949-f99cf5fd3579

Retrieve a document

In this section, we will retrieve the document that was previously written to the database.

  1. Add the following code method to retrieve a document, and again call it from HelloCBL: 

    Document RetrieveDocument (string docID)
{
    // retrieve the document from the database
    var retrievedDoc = db.GetDocument (docID);
    // display the retrieved document
    LogDocProperties (retrievedDoc);
    return retrievedDoc;
}

    static void LogDocProperties (Document doc)
{
    doc.Properties.Select (x => String.Format ("key={0}, value={1}", x.Key, x.Value))
        .ToList ().ForEach (y => Log.Debug (TAG, y));
}

    This code snippet retrieves the document from the database by using the Database class GetDocument() method. The content of the retrieved document is written to the Application Output to log its keys and values. The retrieved document includes the _id and _rev properties created by Couchbase Lite, in addition to the keys and values that were saved in the previous section.

  2. Build and run the app.

  3. View the Application Output to see the contents of the document including the added _id and _rev properties. 

    [CouchbaseEvents] key=name, value=Big Party
[CouchbaseEvents] key=location, value=MyHouse
[CouchbaseEvents] key=date, value=5/19/2015 12:37:32 PM
[CouchbaseEvents] key=_id, value=d1e4a34b-ee7b-4c88-ad0d-fa45c55d7a85
[CouchbaseEvents] key=_rev, value=1-575af7c7479a02c6783033eafbf64176

Update a document

This section of the tutorial code modifies the document that was previously written to the database.

To update an existing document, you will need the key. Let’s take a look: 

void UpdateDocument (string docID)
{
    var doc = db.GetDocument (docID);
    try {
        // Update the document with more data
        var updatedProps = new Dictionary<string, object> (doc.Properties);
        updatedProps.Add ("eventDescription", "Everyone is invited!");
        updatedProps.Add ("address", "123 Elm St.");
        // Save to the Couchbase local Couchbase Lite DB
        doc.PutProperties (updatedProps);
        // display the updated document
        Log.Debug (TAG, "Updated Doc Properties:");
        LogDocProperties (doc);
    } catch (CouchbaseLiteException e) {
        Log.Error (TAG, "Error updating properties in Couchbase Lite database", e);
    }
}

We need to retrieve the original document before we can update its content. Given the Database and docId, we can retrieve the document. The code first makes a new Dictionary object and copies the existing properties of the document into it. Working on a copy of the document properties is helpful because to update a document successfully you need to include the current revision identifier. Then the code adds the value for eventDescription key and address. IMPORTANT: It would be easy to think that all that happens here is that the Document object is updated with data, but in fact this causes that document to be stored in the local database. This is also different from the data being replicated to the Sync Gateway. So far, there is no Sync Gateway involved. We will cover that later. This is currently all local data on the device only.

When a document is updated, Couchbase Lite creates a new revision of the document that contains a new revision identifier in the _rev property. The document identifier in _id always remains the same.

Build and run the app.

The Application Output displays the updated document. The updated document contains the original _id value, a new _rev value, and the new and modified key-value pairs as shown below: 

[CouchbaseEvents] Updated Doc Properties:
[CouchbaseEvents] key=name, value=Big Party
[CouchbaseEvents] key=location, value=MyHouse
[CouchbaseEvents] key=date, value=5/19/2015 12:37:32 PM
[CouchbaseEvents] key=_id, value=d1e4a34b-ee7b-4c88-ad0d-fa45c55d7a85
[CouchbaseEvents] key=_rev, value=2-5748473393a020b5497a6662f8c4d089
[CouchbaseEvents] key=eventDescription, value=Everyone is invited!
[CouchbaseEvents] key=address, value=123 Elm St.

Adding an attachment

In this section we will create an Attachment to the document. Add a method named AddAttachment with the following implementation and call it from HelloCBL. 

void AddAttachment (string docID)
{
    var doc = db.GetDocument (docID);
    try {
        var revision = doc.CurrentRevision.CreateRevision ();
        var text = "This is some text in an attachment";
        var data = Encoding.ASCII.GetBytes(text);
        revision.SetAttachment ("binaryData", "application/octet-stream", data);
        // Save the document and attachment to the local db
        revision.Save ();
    } catch (CouchbaseLiteException e) {
        Log.Error (TAG, "Error saving attachment", e);
    }
}

We start by fetching the document to which we want to attach the binary data. Then we create a byte array to hold our binary (attachment) data. We're simply creating this from a string in this case for demonstration. Then we get the current revision of the document, since the document could have been modified since we last read it, and we want the latest and greatest data. We use the document factory method CreateRevision to request an UnsavedRevision that we wish to update with an attachment. The UnsavedRevision method SetAttachment lets us place the attachment data on the Document.

Note we need to pass three parameters:

  • The name of the attachment (multiple attachments are allowed)
  • The MIME (Multipurpose Internet Mail Extensions) type (see more)
  • The attachment data that will be written to the Database when the UnsavedRevision is saved.

Then we invoke the Save method to save the Document and its attachment to the Database locally.

Once we have a document with an attachment, we will want to retrieve the attachment using the Couchbase Lite API. Let’s see how: 

void ReadAttachment (string docID)
{
    var doc = db.GetExistingDocument (docID);
    var savedRev = doc.CurrentRevision;
    var attachment = savedRev.GetAttachment ("binaryData");
    using (var sr = new StreamReader (attachment.ContentStream)) {
        var data = sr.ReadToEnd ();
        Log.Debug (TAG, data);
    }
}

To obtain the document with the attachment data given the document ID, we call the GetExistingDocument method of the Database instance. From the document, we want to make sure we have the most current revision, and since we saved it previously, it is a SavedRevision instance. As long as we know the unique name of the attachment on that SavedRevision, we retrieve the attachment by name using the GetAttachment method. The attachment's ContentStream property returns the Stream we placed on the document as the actual attachment. We can then read the Stream to obtain the underlying data.

Delete a document

This section of the tutorial code deletes the document that was previously written to the database.

  1. Add the following method and call it from HelloCBL:

    void DeleteDocument (string docID)
{
    try {
        var doc = db.GetDocument (docID);
        doc.Delete ();
        Log.Debug (TAG, String.Format("Deleted document deletion status = {0}", doc.Deleted));
    } catch (CouchbaseLiteException e) {
        Log.Error (TAG, "Cannot delete document", e);
    }
}

    The document is deleted by calling the Delete method on doc. Then to verify the deletion, it logs the value returned by the Deleted property.

  2. Build and run the app.

  3. View the Application Output.

    The output displays a message that says the document was deleted.

    [CouchbaseEvents] Deleted document, deletion status = True
