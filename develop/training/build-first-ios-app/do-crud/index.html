Performing CRUD operations

CRUD operations represent the basic interactions with a Couchbase Lite database. This section demonstrates how to create, retrieve, update, and delete documents. The tutorial code creates just one document and then in turn retrieves it, updates it, and finally deletes it.

In this section, you replace stub code in the HCAppDelegate.m file with functional code.

Create a document

  1. Replace the stub for the createTheDocument method in the HCAppDelegate.m file with the following functional code:

    // creates the document
- (BOOL) createTheDocument {
    
    NSError *error;
    
    // create an object that contains data for the new document
    NSDictionary *myDictionary =
        @{@"message" : @"Hello Couchbase Lite!",
          @"name" : @"Joey",
          @"age" : @15,
          @"timestamp" : [[NSDate date] description]};
    
    // display the data for the new document
    NSLog(@"This is the data for the document: %@", myDictionary);
    
    // create an empty document
    CBLDocument* doc = [_database createDocument];
    
    // save the ID of the new document
    _docID = doc.documentID;
    
    // write the document to the database
    CBLRevision *newRevision = [doc putProperties: myDictionary error: &error];
    if (!newRevision) {
        NSLog (@"Cannot write document to database. Error message: %@", error.localizedDescription);
        return NO;
    }
    
    NSLog(@"Document created and written to database. ID = %@", _docID);
    return YES;
    
}

    The createTheDocument method creates an empty CBLDocument object named doc, which is saved to the database by using the CBLDocument class putProperties:error: method. This method returns a CBLRevision object, which is checked to make sure the document was written successfully.

    NSDictionary objects provide JSON-compatible representations of data that are suitable for creating documents that you can store in the database. The document created by createTheDocument is an NSDictionary object named myDictionary that contains several keys: message, name, age, and timestamp. Each key has a value associated with it. The value for the message key contains the string “Hello Couchbase Lite!”, the value for thetimestamp key contains the time and date the document was created, the value for the name key contains the string "Joey," and the value for the age key contains the integer 15. The document content is written out to the console to log the keys and values.

    When a document is saved to the database, Couchbase Lite generates a document identifier property named _id and a revision identifier property named _rev, and adds them to the stored document.

  2. Build and run the app.

  3. View the console output.

    The console output displays messages similar to the following ones that log the initial document content and the identifier that was assigned to it upon creation.

    2014-05-07 14:25:35.335 HelloCBL[29967:60b] This is the data for the document: {
    age = 15;
    message = "Hello Couchbase Lite!";
    name = Joey;
    timestamp = "2014-05-07 21:25:35 +0000";
}
2014-05-07 14:25:35.337 HelloCBL[29967:60b] Document created and written to database. ID = C132BC15-5A2F-4ECE-9A4A-DA387C2A5B5D

Retrieve a document

  1. Replace the stub for the retrieveTheDocument method in the HCAppDelegate.m file with the following functional code:

    // retrieves the document
- (BOOL) retrieveTheDocument {
    
    // retrieve the document from the database
    CBLDocument *retrievedDoc = [_database documentWithID: _docID];
    
    // display the retrieved document
    NSLog(@"The retrieved document contains: %@", retrievedDoc.properties);
    
    return YES;
}

    The retrieveTheDocument method retrieves the document from the database by using the CBLDatabase class documentWithID: method. The content of the retrieved document is written out to the console to log its keys and values. The retrieved document includes the _id and _rev properties created by Couchbase Lite, in addition to the keys and values written by the createTheDocument method.

  2. Build and run the app.

  3. View the console output.

    The console output displays a message similar to the following one that displays the retrieved document. In the console output you can see the added _id and _rev properties.

    2014-05-07 14:25:35.338 HelloCBL[29967:60b] The retrieved document contains: {
    "_id" = "C132BC15-5A2F-4ECE-9A4A-DA387C2A5B5D";
    "_rev" = "1-4f822c280082da4be3a0b9644762eadb";
    age = 15;
    message = "Hello Couchbase Lite!";
    name = Joey;
    timestamp = "2014-05-07 21:25:35 +0000";
}

Update a document

  1. Replace the stub for the updateTheDocument method in the HCAppDelegate.m file with the following functional code:

    // updates the document
- (BOOL) updateTheDocument {
    
    NSError *error;
    // retrieve the document from the database
    CBLDocument *retrievedDoc = [_database documentWithID: _docID];
    // make a mutable copy of the properties from the document we just retrieved
    NSMutableDictionary *docContent = [retrievedDoc.properties mutableCopy];
    
    // modify the document properties
    [docContent setObject:@"Good Morning Couchbase Lite!!!" forKey:@"message"];
    [docContent setObject:@"breakfast" forKey:@"meal"];
    [docContent setObject:@"Green eggs and ham" forKey:@"entree"];
    [docContent setObject:@"burnt" forKey:@"toast"];
    
    // write the updated document to the database
    CBLSavedRevision *newRev = [retrievedDoc putProperties: docContent error: &error];
    if (!newRev) {
        NSLog (@"Cannot update document. Error message: %@", error.localizedDescription);
    }
    
    // display the new revision of the document
    NSLog (@"The new revision of the document contains: %@", newRev.properties);
    
    return YES;
    
}

    The updateTheDocument method first retrieves the document from the database the same way it was retrieved in the retrieveTheDocument method. It then makes a mutable copy of the document content in an object named docContent. You need a mutable copy to work with because the properties object returned by the CBLDocument class is an NSDictionary object, which is not mutable.

    Next, the updateTheDocument method modifies the document content by changing the value of the message key and adding values for the new keys meal, entree, and toast. The properties in the retrieved document are replaced with the modified content and written to the database via the putProperties:error: method, which returns a CBLSavedRevision object.

    When a document is updated, Couchbase Lite creates a new revision of the document that contains a new revision identifier in the _rev property. The document identifier in _id always remains the same.

  2. Build and run the app.

  3. View the console output.

    The console output displays a message similar to the following one that displays the updated document. The updated document contains the original _id value, a new _rev value, the new and modified key-value pairs, and the unmodified key-value pairs. 

    2014-05-07 14:25:35.341 HelloCBL[29967:60b] The new revision of the document contains: {
    "_id" = "C132BC15-5A2F-4ECE-9A4A-DA387C2A5B5D";
    "_rev" = "2-f4d667aabdb483ec34f6e989ac4a36e1";
    age = 15;
    entree = "Green eggs and ham";
    meal = breakfast;
    message = "Good Morning Couchbase Lite!!!";
    name = Joey;
    timestamp = "2014-05-07 21:25:35 +0000";
    toast = burnt;
}

Delete a document

  1. Replace the stub for the deleteTheDocument method in the HCAppDelegate.m file with the following functional code:

    // deletes the document
- (BOOL) deleteTheDocument {
    
    NSError *error;
    
    // retrieve the document from the database and then delete it
    if (![[_database documentWithID: _docID] deleteDocument: &error])
        NSLog (@"Cannot delete document. Error message: %@", 
            error.localizedDescription);
    // verify the deletion by retrieving the document and 
    // checking whether it has been deleted
    CBLDocument *ddoc = [_database documentWithID: _docID];
    NSLog (@"The document with ID %@ is %@", _docID, 
        ([ddoc isDeleted] ? @"deleted" : @"not deleted"));
    
    return YES;
    
}

    The deleteTheDocument method retrieves the document from the database to get a pointer to it, and then deletes it via the deleteDocument: method from the CBLDocument class. After the document is deleted, the method retrieves the document again to verify that it was deleted and logs a message regarding the deletion status.

  2. Build and run the app.

  3. View the console output.

    The console output displays a message similar to the following one that says the document was deleted.

    2014-05-07 14:25:35.343 HelloCBL[29967:60b] The document with ID C132BC15-5A2F-4ECE-9A4A-DA387C2A5B5D is deleted
